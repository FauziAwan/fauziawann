9 Manfaat Komentar pada Kode Program yang Jarang diketahui

Ardianta Pargo

Komentar memang akan diabaikan oleh kompilator dan interpreter. Tapi sangat berguna bagi programmer.

Menurut Wikipedia,

Dalam pemrograman komputer, sebuah komentar adalah penjelasan atau anotasi dalam kode program yang bisa dibaca oleh programmer (programmer-readable). Komentar dibuat dengan tujuan agar kode program mudah dipahami manusia.

Saya sendiri termasuk orang yang jarang menulis komentar, apalagi komentar yang panjang.

(nulis kode aja masih copas, apalagi mau nulis komentar ðŸ˜„)

Tapi tahukah kalian, apa manfaat dan bagaimana penggunaan komentar dalam kode program?

Mari kita bahas….

1. Membuat Rencana dan Review

Kadang saat kita membuat kode untuk logika tertentu, kita membuat komentarnya dulu baru setelah itu membuat kodenya.

Artinya nanti kita akan berenacan membuat logikanya seperti yang dituliskan pada komentar.

Contoh: Recanan untuk membuat logika login

// 1. ambil username dan password
// 2. Cek username, apakah ada di database
//      - kalau ada cek passwordnya
//      - kalau tidak kasih tahu kalau dia belum terdaftar
// 3. alihkan ke halaman dashboard kalau berhasil login
// 4. tetap berada di halaman login kalau gagal

Rencana akan memudahkan kita menulis kode ke depannya. Jadi, cobalah untuk melakukannya.

Sedangkan untuk review, kita menulis komentar terhadap kode yang sudah ada agar mudah dipahami.

Contoh:

/* loop backwards through all elements returned by the server (they should be processed chronologically)*/
for (i = (numElementsReturned - 1); i >= 0; i--){
    /* process each element's data */
    updatePattern(i, returnedElements[i]);
}

Kadang dalam membuat rencana dan review dilakukan dengan menggunakan tag seperti: @TODO, @FIXME, @HACK, dll.

// @TODO buat kode untuk pendaftaran user baru
// @FIXME bugs error 404

Pada teks editor atom, tag tersebut akan diwarnai.
