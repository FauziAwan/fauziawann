Cara Menggunakan DotEnv untuk Mengamankan Konfigurasi Web

Biasanya kita menyimpan konfigurasi web pada file config.php, config.db.php, database.php, dsb.

Sebenarnya cara ini boleh-boleh saja selama tidak menggunakan Git dan source code-nya tertutup.

Beberapa waktu yang lalu, saya pernah membahas tentang pencurian password di Github. Siapa saja bisa dengan mudah mencuri password dari server melalui catatan commit.

Kalau hanya password localhost yang dicuri, kita tidak perlu khawatir. Tapi, bagaimana kalau password server production atau server pemerintah?

Lalu kita belum sempat mengganti passwordnya, karena si pencuri sudah duluan mengambil alih servernya.

Bisa jadi masalah besar…

Lalu bagaimana solusinya?

Gunakan variabel environment (lingkungan).

Apa itu Variabel Environment?

Variabel Environment merupakan variabel yang ada di setiap komputer atau server. Nilainya bisa berbeda-beda di setiap server.

Contoh variabel Environment di Linux:

  • $HOME — variabel yang menyimpan alamat home;
  • $PATH — variabel yang menyimpan lokasi bin;
  • $HOSTNAME — variabel yang menyimpan nama komputer/host.

Cara mengaksesk variabel Environment di Linux: buka terminal dan coba ketik perintah echo $HOME.

![Melihat isi variabel env $HOME]()

Cara membuat variabel Environment baru:

export nama=petanikode

Kemudian coba echo variabel tersebut:

echo $nama

Maka akan menghasilkan output: petanikode.

Tapi…saat kita tutup terminal-nya, variabelnya menghilang.

Bagaimana agar tidak menghilang?

Agar tidak menghilang, tulis perintah export tersebut di dalam file ~/.bashrc.
