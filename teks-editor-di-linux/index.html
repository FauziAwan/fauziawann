Review & Rekomendasi 9 Teks Editor di Linux untuk Menulis Kode (Pemrograman)

Teks editor adalah aplikasi wajib yang harus diinstal oleh programmer. Tanpanya, programmer tidak bisa menulis kode.

Jika kamu mencari di Google dengan kata kunci “Text editor Linux” maka Google akan menampilkan beberapa teks editor yang didukung di Linux.

Banyak yang masih bingung menetukan teks editor mana yang bagus untuk pemrograman. Ada juga yang belum tahu sama sekali.

Karena itu…Pada tulisan ini, saya akan melakukan review terhadap beberapa teks editor yang pernah saya coba di Linux dan memberikan rekomendasi berdasarkan pengalaman saya.

Teks editor yang akan dibahas adalah teks editor berbasis GUI yang umum digunakan untuk pemrograman.

Untuk yang berbasis teks seperti VIM, silahkan baca: 6 Teks editor berbasis teks di Linux.

Baiklah, mari kita mulai…

Berikut ini 10 Teks editornya:

1. Sublime Text

Siapa yang tidak kenal dengan teks editor yang satu ini.

Teks editor sejuta umat… ðŸ˜„

Sublime Text adalah teks editor ke-2 saya setelah Notepad++.

Text Editor ini cukup handal digunakan untuk berbagai macam bahasa pemrograman.

Sublime adalah software proprietary (berbayar), namun bisa diinstal di Linux.

Kita diberikan masa tertentu untuk mencoba (trial), setelah itu… untuk menggunakannya secara full, kita harus membayar lisensinya.

Kamu bisa men-dwonload Sublime Text di website resminya: [Download Sublime].

2. Atom

Teks editor Atom hadir untuk bersaing dengan Sublime.

“A hackable text editor for the 21st Century”

Begitulah bunyi selogannya…

Atom dibangun dengan bahasa pemrograman Javascript dan menggunakan framework Electron.

Teks editor atom cukup bagus, tapi agak berat dibandingkan Sublime.

Membuka Teks Editor Atom sama beratnya seperti membuka google chrome

— Petani Kode (@petanikode) June 21, 2017

Buat kamu yang punya memori cukup banyak, bisa coba Text Editor ini.

Kamu bisa download Atom di website reseminya: atom.io

Ada beberapa masalah yang saya temukan saat menggunakan atom:

  1. Pada Linux, Atom kadang tidak bisa menghapus direktori/file dari source tree.
  2. Berat, memakan banyak memeori dan kadang-kadang hang.
  3. Autocomplete-nya kadang membut typo ðŸ˜„.

3. Visual Studio Code

Teks Editor opensouce racikan Microsoft ini bisa jadi alternatif bila tidak ingin menggunakan Atom maupun Sublime.

Sama-sama dibuat dengan Electorn seperti Atom, tapi menurut saya VS Code lebih ringan daripada Atom.

Selain itu, VS Code juga sudah terintegrasi dengan Git dan di dalamnya terdapat terminal yang bisa dibuka kapan saja.

VS Code memiliki banyak tema dan plugin yang melengkapinya.
